# 🎧 데이터 분석 부트캠프 12일차
## Python EDA 실습: Spotify 인기곡 특성 분석

데이터 분석은 단순히 코드를 작성하고 결과를 확인하는 과정이 아니라,

**"어떤 문제를 해결하기 위해 데이터를 분석할 것인가"**

에서 시작한다고 생각한다.

오늘은 Spotify 음악 데이터를 활용하여

> **"인기 있는 음악은 어떤 특징을 가지고 있을까?"**

라는 질문에서 출발해 탐색적 데이터 분석(EDA)을 진행했다.

이번 포스팅에서는 단순히 실습 내용을 정리하는 것이 아니라,

- 어떤 데이터를 활용했는지
- 어떤 기준으로 데이터를 바라봤는지
- 어떤 분석 과정을 거쳤는지
- 결과를 어떻게 해석했는지

를 데이터 분석가의 관점에서 기록해보고자 한다.

---

## 🎯 1. 분석 목적 (Problem Definition)

좋은 데이터 분석은 데이터 탐색보다 먼저 **좋은 질문을 정의하는 것**에서 시작한다.

Spotify 데이터를 바라보고 생각나는 질문을 정리해보았다.

아래 질문들에 답을 찾아가며 분석 목적에 다가가고자 한다.

- 인기도 점수 분포는?
- 인기도 점수에 영향을 미치는 요인은? → 인기도 점수와 변수 간의 상관관계?
- `explicit` 요소가 포함된 노래가 더 인기가 있을까? → 자극적인 콘텐츠 소비 경향과 관련이 있을지 파악해보자!
- 장르별 음악 소비 패턴 차이가 있을 것이다?
- 인기곡과 비인기곡의 차이는 무엇일까? → 인기도 점수 기준 상하위 10%곡을 뽑아 인기곡/비인기곡 으로 나누어 특성을 분석해보자!
- 시끄러운 곡이거나 신나는 곡이 조용한 곡보다 인기가 많을까?

## 📁 2. 데이터 소개 (Dataset)

## 데이터 출처

코드잇 부트캠프에서 제공한 Spotify 음악 데이터를 활용했다.

데이터에는 곡의 기본 정보 뿐만 아니라 음악적 특징을 나타내는 다양한 변수가 포함되어 있었다.

## 주요 컬럼

| 컬럼 | 설명 |
| --- | --- |
| track_id | 고유 ID |
| popularity | 인기도 점수 |
| duration_ms | 길이 (ms) |
| explicit | 욕설이나 선정적/폭력적/차별적 표현 포함 여부 |
| danceability | 춤추기 적합한 정도 |
| energy | 강도와 활동성 |
| key | 키 |
| loudness | 음량_데시벨 (db) |
| mode | 음계 (장조 or 단조)|
| speechiness | 말하는 부분 포함 정도 |
| acousticness | 어쿠스틱 특성을 가진 정도 |
| instrumentalness | 기악곡일 가능성 |
| liveness | 라이브 관객 존재감 정도 |
| valence | 긍정성의 정도 |
| tempo | 템포를 분당 박자수(BPM)로 나타낸 값 |
| time_signature | 각 마디 내 박자 수 (4/4박자 등)|
| track_genre| 장르 |

음악에 포함된 다양한 요소들이 수치 데이터로 포함되어 있다는 점이 흥미로웠다.

## 🧹 3. 데이터 전처리 (Data Preprocessing)

잘못된 데이터로 분석 시, 왜곡된 분석 결과를 초래할 수 있기 때문에 데이터 전처리 과정은 필수적이다.

가장 먼저 결측치, 중복값, 이상치 등을 확인했다.

1. 데이터 기본 정보
    - df.shape
    - df.info()
    - df.columns
2. 결측치
    - df.isnull().sum()
3. 중복값
    - df.duplicated().sum()
    - df.drop_duplicates()
4. 이상치
    - df.describe()

---

### 데이터 전처리 과정에서 배운 점

실제 데이터는 지금보다 더 비정제된 형태로 주어질 것이다.

절대 분석하기 좋은 형태로 제공되지 않을 것이다.

따라서 데이터 분석가는 **분석 목적에 맞게 데이터를 정제**할 수 있어야 한다.

## 🔎 4. 탐색적 데이터 분석 (EDA)

EDA의 목적은 단순히 그래프를 그려 시각화하는 것이 아니다.

물론 탐색한 결과를 쉽게 파악하기 위해 그래프 시각화는 필수적으로 하면 좋다.

데이터 속에서
- 어떤 패턴이 존재하는가?
- 이 패턴은 어떤 의미를 가지는가?
- 추가적으로 확인해야 할 부분은 무엇인가?

를 발견하는 것이 중요한 과정이라고 할 수 있다.

---

### 📊 4-1. Popularity 분포 확인

먼저 인기도 점수(`popularity`)의 전체적인 분포를 **히스토그램(histplot)**으로 파악했다.

```
plt.figure(figsize=(20, 5))

sns.histplot(data=df, x='popularity',
             kde=True,
             stat='density',
             palette='Blues_r')
plt.show()
```

👉 인기도 점수 0점 부근에 많은 곡들이 포진해 있고, 10~60점 정도까지는 거의 고르게 분포하고 있었다.


또, 몰랐던 지식인 `stat='density'` 옵션에 대해 배웠다.
> **❓ 왜 '밀도(density)'로 바꿔서 보나요?**
> 
> - 데이터의 전체 개수가 서로 다른 두 집단(예: 전체 곡 vs 특정 장르 곡)의 분포 모양을 **공정하게 비교**하고 싶을 때 사용합니다. 
> 
> - 개수로만 그리게 되면 데이터 양이 많은 집단의 막대만 무조건 높게 솟구치기 때문에, 면적을 1로 통일(`stat='density'`)해 주어야 곡의 개수와 상관없이 **"어느 구간에 상대적으로 더 많이 몰려있는지"** 분포의 형태를 올바르게 비교할 수 있습니다.

---

### 🎵 4-2. 인기도 점수와 기타 숫자형 변수(음악적 특성을 나타내는 각 변수의 정도를 수치로 나타낸 값)

음악 특성을 나타내는 다양한 수치형 변수들이 popularity와 어떤 연관성을 가지는지 확인하기 위해 상관분석을 진행했다.

각 변수 간 상관계수를 계산한 뒤, 계산한 상관관계 행렬을 **히트맵(heatmap)**으로 시각화하여 변수 간 관계의 패턴을 파악했다.

```
numeric_cols = df.select_dtypes(include='number').columns

corr_matrix = df[numeric_cols].corr()
sns.heatmap(corr_matrix, annot=True, fmt='.2f', cmap='vlag', vmin=-1, vmax=1)

plt.show()
```

단순히 높은 상관관계를 가진 변수를 찾는 것보다,

> "왜 이러한 관계가 나타났을까?"

를 고민하는 것이 중요하다는 것을 깨달았다.

각 요인의 특성과 인기도 점수의 상관관계를 고려해 판단해야 하는데,

그걸 위해 "도메인 지식"이 필요하다고 느꼈다.

---

### 🎼 4-3. 장르별 인기 차이 분석

장르별 popularity 차이를 비교했다.

분명히, 장르별로는 인기도에 차이가 있을 것이라고 생각하고 분석을 진행했다.

분석 결과,  
pop, k-pop과 같이 대중성이 높은 장르는 높은 인기도를 보였으며,  
jazz, soul과 같이 상대적으로 대중성이 낮은 장르는 낮은 인기도를 보이는 경향이 있었다.

오늘은 부트캠프에서 진행한 실습이기에 더 깊이 분석을 진행하진 않았지만,

만약 이게 실제 데이터 분석 프로젝트였다면,

- 시대적 트렌드
- Spotify 유저의 특성(취향, 나이, 성별 등)
- 아티스트 영향력

등 다양한 요인이 함께 영향을 줬을 것이라 생각하고, 추가 분석을 진행해야 할 것이라 생각했다.

## 💡 5. 분석 결과 및 인사이트

이번 분석을 통해 가장 크게 느낀 점은,

> **데이터 분석은 결과를 찾는 과정이 아니라,**  
> **탐색적 데이터 분석(EDA)을 통해 내가 던진 질문에 답을 찾고,**  
> **결과를 해석하며 인사이트를 도출하는 과정이다.**  

라는 것이다.

📌 깨달은 점:  
✅ 데이터 전처리는 분석 전 필수적으로 진행해야 한다.  
✅ 시각화는 데이터 속 패턴을 발견하는 중요한 도구다.  
✅ 상관관계와 인과관계를 구분해 변수별 특성을 고려해 분석해야 한다.  
✅ 분석가는 코드의 결과를 넘어서 그 뒤에 존재하는 의미를 찾아 고민해야 한다.

## 🚧 6.한계점 및 개선 방향

이번 분석은 주어진 Spotify 데이터 속 변수만을 활용해야 했기 때문에 한계가 있었다.

음악의 인기도를 생각하면 단순한 음악적 특성 뿐만 아니라 플랫폼과 유저의 특성 등 다양한 외부 요인의 영향이 있을 것이다.

- 유저 데이터 추가
- 유저별 플레이리스트 데이터 분석
- 발매 연도별 인기 변화도 패턴 파악 (시대별 트렌드 분석)
- 국가별 음악 소비 패턴 비교
- 발매 예정 곡의 인기도 예측 머신러닝 모델 구축

등을 진행해보고 싶다.
